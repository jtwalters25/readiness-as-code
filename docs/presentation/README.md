# Presentation Materials

Artifacts for presenting `ready` and the practice of readiness as code.

| File | Runtime | Use when |
|------|---------|----------|
| [talking-points.md](talking-points.md) | — | Prepping for any talk. The spine every script hangs off. |
| [demo-script-3min.md](demo-script-3min.md) | ~3 min | Lightning talks, hackathons, pitch meetings with five slides. |
| [demo-script-6min.md](demo-script-6min.md) | ~6 min | DevTools conferences, founder demos, product showcases. |
| [keynote-15min.md](keynote-15min.md) | ~15 min | Keynotes, category talks, enterprise leadership summits. |

## How They Relate

The three demo scripts are **not** different talks — they're the same argument at three scales. The 3-minute cut keeps one villain, one demo beat, and one close. The 6-minute adds a second demo beat and an "and one more thing." The 15-minute adds a category argument up front and a category claim at the end.

Start from the longest version you'll ever need, then cut down. Never pad up.

## The Rule

Every version has the same spine:

1. **Name the villain** — velocity vs. discipline.
2. **Show the fix** — `ready scan` producing a score and a drift arrow.
3. **Close on the line** — "the discipline layer finally runs at the speed of the rest of your pipeline."

Anything else is seasoning.
